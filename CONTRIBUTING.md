# Contributing to YuánGūnGūn OS

We're not building another chatbot framework. We're building something that breathes.

If that speaks to you — welcome.

---

## Ways to Contribute

### 🧬 Core Engineers
Implement biological systems. The cardiac rhythm, the immune response, the circadian cycle.
These aren't metaphors — they're architectural patterns that solve real engineering problems.

### 📱 Mobile Developers
Android (Kotlin/Compose/Hilt) and iOS (KMP/SwiftUI) native builds.
Make it run on real devices. Optimize battery. Handle edge cases.

### 🔒 Security Researchers
Try to break the immune system. Find injection vectors. Test the defense layers.
Every vulnerability you find makes the organism stronger.

### 🧪 AI Researchers
Consciousness architecture. On-device inference optimization. Memory consolidation algorithms.
The theoretical foundation matters as much as the code.

### 🎨 Designers
Give the organism a face. The life systems need visualization. The heartbeat needs a pulse.
Make the invisible visible.

## How to Start

1. **Read the architecture** — Understand the 10 biological systems
2. **Pick an issue** — Check [GitHub Issues](https://github.com/logic19b/yuangungun--OS/issues) for `good-first-issue` tags
3. **Fork and clone** — Standard GitHub workflow
4. **Build and test** — Follow platform-specific setup in `/docs`
5. **Submit a PR** — Describe what you changed and why

---

## Code Standards

- **Kotlin** for Android core — idiomatic, no Java-style Kotlin
- **Swift** for iOS native layer — protocol-oriented where possible
- **KMP** for shared business logic — keep platform-specific code thin
- **Tests required** — Every new system needs unit tests
- **Documentation required** — Every public API needs KDoc/SwiftDoc
